# いつものやつ（仮称）— 仕様書 v0.1

> 普段使っている商品のパッケージ写真を家族・グループで共有し、お使いのときに
> 「いつものやつ」を正確に買ってきてもらうためのアプリ。

## 1. 背景・目的

お使いを頼むとき、頼まれた人が「いつも使っているもの」を正確に把握していないため、
おおむね同じだが別メーカー・別仕様の商品を買ってきてしまいトラブルになることがある。

本アプリは、普段使っている商品の **パッケージ写真** と **メーカー・仕様などの情報** を
記録・共有し、お使いの際に写真でそのまま伝えられるようにすることでこの問題を解決する。

## 2. 主な利用シーン

1. **登録**: 家にある「いつものやつ」を撮影し、商品名・メーカー等を添えて登録する。
2. **共有**: 家族グループに招待し、同じリストを全員が閲覧・編集できるようにする。
3. **お使い依頼**: 「今日買ってきてほしいもの」を登録済み商品から選んでお使いリストを作る。
4. **店頭**: 頼まれた人がお使いリストを開き、各商品のパッケージ写真を見ながら正しい商品を買う。

## 3. 用語

| 用語 | 意味 |
|---|---|
| 商品 (Item) | 「いつものやつ」1点。パッケージ写真とメタ情報を持つ。 |
| グループ (Group) | 商品リストを共有する単位（家族など）。 |
| メンバー (Member) | グループに属するユーザー。 |
| お使いリスト (ShoppingList) | 「買ってきてほしいもの」を商品から選んで作る買い物リスト。 |

## 4. 機能要件（MVP）

### 4.1 商品の登録・一覧・詳細
- **登録**: パッケージ写真（1枚以上、複数可）、商品名、メーカー/ブランドを必須に、
  カテゴリ・購入場所・容量/規格・メモを任意で入力できる。
  - 写真はスマホのカメラ撮影またはライブラリ選択（PWA: `<input type="file" capture>`）。
- **一覧**: 同じグループの商品をカード（写真＋商品名＋メーカー）で一覧表示。
  - カテゴリでの絞り込み、商品名/メーカーでのキーワード検索。
- **詳細**: パッケージ写真を大きく表示（店頭で見比べやすく）。全メタ情報を表示。
- **編集・削除**: 登録した商品を編集・削除できる。
- **在庫フラグ（任意・軽量）**: 「在庫切れ」マークを付けられる。

### 4.2 グループ共有
- ユーザーはグループを作成でき、招待リンク等で他メンバーを招待できる。
- グループ内の商品・お使いリストは全メンバーが閲覧・編集できる。
- 1ユーザーが複数グループに所属できる（例: 自宅と実家）。

### 4.3 お使いリスト機能
- 登録済み商品から選んでお使いリストを作成する（数量・メモ付き）。
- 頼まれた人はリストを開き、各項目の写真を見ながら買い物できる。
- 各項目に「買った」チェックを付けられる。
- 担当者（誰に頼んだか）を任意で設定できる。

## 5. 画面一覧・主要フロー

| 画面 | 内容 |
|---|---|
| ログイン/サインアップ | アカウント作成・ログイン |
| グループ選択/作成 | 所属グループの切替・新規作成・招待 |
| 商品一覧 | 検索・カテゴリ絞り込み・商品カード一覧 |
| 商品詳細 | 大きな写真＋メタ情報、編集/削除、お使いに追加 |
| 商品登録/編集 | 写真撮影＋情報入力フォーム |
| お使いリスト一覧 | 進行中/完了のリスト一覧 |
| お使いリスト詳細 | 項目＋写真、チェック、担当者 |

**登録フロー**: 一覧 →「＋」→ 撮影 → 情報入力 → 保存 → 一覧に反映
**お使いフロー**: 商品を選択 → お使いリストに追加 → 共有 → 店頭でチェック

## 6. データモデル（案）

```
User
  id, displayName, email, photoURL, createdAt

Group
  id, name, ownerId, createdAt
  members: [{ userId, role: owner|member, joinedAt }]

Item                      # 商品（グループに属する）
  id, groupId
  name            (必須)  # 商品名
  brand           (必須)  # メーカー/ブランド
  category        (任意)  # 洗剤/食品/日用品 ...
  store           (任意)  # 購入場所
  size            (任意)  # 容量・規格
  note            (任意)  # メモ
  jan             (任意)  # バーコード（将来の照合用に枠だけ確保）
  photos: [ { url, isPrimary } ]   # 1枚以上
  inStock         (任意)  # 在庫フラグ
  createdBy, createdAt, updatedAt

ShoppingList              # お使いリスト
  id, groupId, title
  status: active|done
  assigneeId      (任意)  # 頼んだ相手
  createdBy, createdAt
  items: [ ShoppingListItem ]

ShoppingListItem
  id, itemId              # Item への参照
  quantity, note
  checked: boolean
```

## 7. 技術構成（案）

Web/PWA × クラウド同期を前提とした構成案。

| 層 | 候補 | 補足 |
|---|---|---|
| フロント | React + Vite（PWA対応） | TypeScript。インストール不要で試せる。 |
| 認証 | Firebase Auth または Supabase Auth | Google ログイン等 |
| DB | Firestore または Supabase(Postgres) | リアルタイム同期が容易 |
| 画像保存 | Cloud Storage / Supabase Storage | アップロード時にリサイズ |
| ホスティング | Firebase Hosting / Vercel | |

> 認証・DB・ストレージ・リアルタイム同期が一体な **Firebase** か **Supabase** の
> どちらかを推奨。次の設計フェーズで確定する。

## 8. 非機能要件

- **モバイル最優先**: スマホ縦画面のUIを基準に設計。
- **オフライン**: PWAで一覧・写真の閲覧は可能な限りキャッシュ（店頭で電波が弱くても見られる）。
- **画像最適化**: アップロード時に圧縮・サムネイル生成し、一覧は軽量化。
- **プライバシー**: グループ外には商品・写真を公開しない。
